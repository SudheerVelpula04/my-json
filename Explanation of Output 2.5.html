Code:

javascript
fetch("http://localhost:3000/data")
  .then(res => console.log(res))
  .catch(err => console.log(err));

🔎 Step 1: fetch() Request

  • The fetch() method is used to make an HTTP request.
  • It sends a GET request to the provided URL (http://localhost:3000/data).
  • The fetch() method returns a Promise.

🔎 Step 2: .then() Block

  • If the request is successful (even if the status code is 404, 500, etc.), the .then() block will execute.
  • The res object is the Response object, not the actual data.
    The Response object contains information like:
    • res.ok → true if the status is 200-299 (success range)
    • res.status → The HTTP status code (e.g., 200, 404, 500)
    • res.json() or res.text() → Methods to read the actual response body

Since you're logging res directly, it will print an object like this:

lua
Response { type: "cors", url: "http://localhost:3000/data", status: 200, ok: true, ... }

To view the actual data, you need to call .json() or .text() like this:

javascript
fetch("http://localhost:3000/data")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.log("Error:", err));

🔎 Step 3: .catch() Block

  • The .catch() block will only run if:
    • The fetch() request fails to connect (e.g., server is down, URL is wrong).
    • Network errors (e.g., no internet connection).
  • It will not catch HTTP errors like 404 or 500 — those still count as successful requests from fetch()'s perspective.

🚨 Common Issues and Solutions

  1. Server Not Running
    → Error: TypeError: Failed to fetch
    🔹 Ensure your server at localhost:3000 is active.

  2. Incorrect URL
    → Error: TypeError: Failed to fetch
    🔹 Double-check the endpoint path.

  3. Unexpected Data Format
    → Error in .then(res => res.json())
    🔹 Ensure the API response is actually JSON.

✅ Correct Example for Fetching JSON Data

javascript
fetch("http://localhost:3000/data")
  .then(res => {
    if (!res.ok) {
      throw new Error(`HTTP error! Status: ${res.status}`);
    }
    return res.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error("Error:", err));

This ensures:

  • Proper handling of HTTP errors.
  • Clean display of the actual data.

Here's a simple Python code that:

✅ Accepts Name, Email, and Password from the user.
✅ Stores the data securely in a .json file.

🔒 Code for Storing User Data in a JSON File

python
import json
import os

# Function to add user data
def add_user():
    name = input("Enter your name: ")
    email = input("Enter your email: ")
    password = input("Enter your password: ")

    # Create a dictionary to store user data
    user_data = {
        "name": name,
        "email": email,
        "password": password
    }

    # Check if file exists; if yes, load existing data
    if os.path.exists("user_data.json"):
        with open("user_data.json", "r") as file:
            try:
                data = json.load(file)
            except json.JSONDecodeError:
                data = []  # Empty list if file is corrupted or empty
    else:
        data = []

    # Append the new data and write back to JSON file
    data.append(user_data)

    with open("user_data.json", "w") as file:
        json.dump(data, file, indent=4)

    print("✅ User data stored successfully!")

# Run the function
add_user()
